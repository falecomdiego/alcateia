# Versões substituídas — MDN-RPP01

Quando um documento versionado for substituído:

1. copiar a versão anterior para esta pasta sem alterar seu conteúdo;
2. manter código e versão no nome;
3. atualizar o índice de versões;
4. registrar decisão, justificativa e impacto;
5. nunca usar apenas os termos `final`, `final_novo` ou `final_corrigido`.

Logs históricos não devem ser reescritos. Correções recebem nova linha ou novo artefato vinculado.
