# Diretrizes Aprovadas do Manual de Identidade Visual do Mapa da Noite

## 1. Status

Documento operacional derivado do Manual de Identidade Visual em construção.

Aplicar apenas regras aprovadas e proibições vigentes.

Este documento não substitui o manual completo em desenvolvimento e não deve ser tratado como Manual Final.

## 2. Finalidade

Orientar a criação de cards, carrosséis, stories, painéis, mockups, CTAs, imagens de divulgação, materiais digitais e demais peças visuais do projeto Mapa da Noite, usando apenas diretrizes já aprovadas.

Este recorte operacional existe para dar segurança de execução enquanto o Manual de Identidade Visual completo ainda está em construção.

## 3. Diretrizes gerais aprovadas

- Buscar acabamento de alto padrão.
- Atuar com direção de arte sênior.
- Construir peças com lógica de campanha, não apenas como posts isolados.
- Priorizar clareza, sofisticação e inteligência coletiva.
- Trabalhar sempre a partir de dados reais, comentários públicos e evidências auditáveis.
- Usar linguagem urbana, noturna, cartográfica, editorial, tecnológica e publicitária.
- Preservar baixa carga cognitiva.
- Criar hierarquia visual clara entre título, dado, explicação e encaminhamento.
- Usar elementos visuais de forma funcional, não decorativa.
- Respeitar recuo de segurança.
- Criar tensão narrativa inicial quando fizer sentido.
- Construir relação coerente entre texto, layout e narrativa.

## 4. Diretrizes visuais aprovadas

- Usar Google Sans como referência tipográfica.
- Usar amarelo como cor de assinatura.
- Usar preto, grafite e navy como base estrutural.
- Usar off-white, cinza-gelo, prata urbano e bases frias claras.
- Usar grafite quente, vinho profundo, marrom queimado, cobre e âmbar como fundos escuros ou quentes.
- Usar verde escuro apenas como acento técnico mínimo.
- Alternar bases claras/frias e escuras/quentes.
- Priorizar mockups, perspectiva 3D, profundidade, camadas e open design.
- Usar fotos realísticas, imagens realísticas, mockups realísticos, ambientes reais e representações fotográficas.

## 5. Diretrizes editoriais aprovadas

- Usar storytelling.
- Abrir com pergunta estratégica, tensão narrativa ou fricção cognitiva quando isso fortalecer a peça.
- Manter clareza para público leigo.
- Preservar densidade sem superficialidade.
- Distinguir tensão narrativa de CTA.
- Conduzir o leitor por progressão narrativa.
- Mostrar comentários reais transformados em evidência.
- Usar dados como elemento de autoridade.

## 6. Regras aprovadas para carrossel

- Todo carrossel deve ter storytelling.
- Cada card deve cumprir função narrativa clara.
- A abertura deve trazer pergunta ou tensão estruturante quando fizer sentido.
- A sequência deve ter progressão narrativa.
- Alternar visualmente os cards para evitar repetição de fundo, cenário ou composição-base.
- Preservar baixa carga cognitiva.
- Respeitar recuo de segurança.
- Dar destaque visual a números, datas, períodos, quantidades e indicadores.
- Usar ícones contextuais quando fizer sentido.
- Priorizar mockups, profundidade, perspectiva 3D e open design.
- Buscar aparência de campanha publicitária de alto padrão.
- Usar fotos e representações realísticas.

## 7. Regras aprovadas para redes sociais e stories

- Priorizar leitura mobile.
- Respeitar recuo de segurança.
- Usar CTA claro quando necessário.
- Destacar dados relevantes.
- Preservar contraste.
- Usar mensagem curta em stories.
- Usar proporção 9:16 para stories.
- Abrir com frase que gere curiosidade, identificação ou deslocamento perceptivo.

## 8. Regras aprovadas para painéis

- Painel interno deve funcionar como ferramenta real, funcional e auditável.
- Painel interno deve preservar rastreabilidade, classificação, evidências, recomendações e status da análise.
- Dashboard deve usar cards, tabelas, gráficos, indicadores e hierarquia clara.
- Painel público deve ser camada acessível, menos técnica, com dados reais, auditáveis e mensuráveis.
- Painel público deve reduzir complexidade sem perder aderência aos dados.

## 9. Ícones e elementos gráficos aprovados

- Ícones devem ter função semântica.
- Usar calendário para datas e períodos.
- Usar avatar, perfil ou grupo para quantidades de perfis, contas, pessoas ou grupos.
- Usar balão de fala, comentário, banco de dados, radar, busca ou evidência para comentários e amostras.
- Usar linhas, pontos, coordenadas, cantos, marcadores, ondas, barras e elementos de interface com sobriedade.
- Usar São Paulo, noite, cartografia, skyline, mapa, coordenadas, luzes e traçados urbanos como linguagem visual.

## 10. Modo operacional aprovado de interação com IA

- Consultar as diretrizes aprovadas antes de criar imagem, card, carrossel, story, painel ou prompt visual.
- Atuar como diretor de arte sênior.
- Seguir o fluxo obrigatório: conteúdo do usuário → análise → cruzamento com diretrizes aprovadas → apresentação de prompt ou direção criativa → aprovação do usuário → geração apenas após comando explícito.
- Avaliar tensão narrativa.
- Sugerir abertura mais forte quando o texto estiver apenas informativo.
- Antecipar riscos.
- Realizar checkpoint automático de consistência por volta do quarto ou quinto card.

## 11. Proibições vigentes

- Não gerar imagem sem autorização explícita.
- Não tratar sugestão, referência, ajuste ou aprovação parcial como autorização para gerar imagem.
- Não usar blocos translúcidos, caixas translúcidas, overlays neon, painéis semitransparentes, glassmorphism ou recursos similares.
- Não usar neon excessivo, magenta dominante, cyberpunk genérico ou estética hacker.
- Não usar HQ, cartoon, personagens caricaturais ou estética infantilizada.
- Não usar textos próximos às bordas.
- Não repetir imagem de fundo, cenário ou composição-base dentro do mesmo carrossel.
- Não expor dados pessoais, nomes sensíveis, caminhos de arquivos ou prints brutos.
- Não reproduzir literalmente interfaces proprietárias.
- Não usar cores sem função semântica.
- Não usar fontes aleatórias.
- Não usar fragmentação textual excessiva.
- Não tornar painel público excessivamente técnico ou poluído.

## 12. Itens descartados

Os itens abaixo ficam registrados apenas como bloqueio histórico e não devem ser retomados como diretriz válida:

- preto e amarelo em todos os cards;
- blocos translúcidos e neon;
- fundo branco plano sem profundidade;
- Montserrat e Inter como sistema oficial.

## 13. Itens não aplicáveis automaticamente neste momento

Itens [Em validação], [Pendente] e “itens de avaliação futura” permanecem fora da aplicação automática até aprovação explícita do usuário. Eles podem ser consultados como pauta futura, mas não devem orientar produção imediata.

## 14. Retomada rápida

- Usar Google Sans.
- Usar amarelo como assinatura.
- Alternar bases claras/frias e escuras/quentes.
- Priorizar mockups, profundidade, perspectiva 3D e open design.
- Usar imagens realísticas.
- Iniciar com tensão narrativa sempre que possível.
- Respeitar recuo de segurança.
- Não usar HQ, cartoon, glassmorphism, neon excessivo ou blocos translúcidos.
- Não gerar imagem sem comando explícito.
- Realizar checkpoint na metade de carrosséis.

## 15. Blindagem operacional

Este documento é apenas documentação das diretrizes visuais aprovadas.

Ele não executa análise, não reprocessa comentários, não altera dados, não altera matriz, não altera painel e não gera imagem.
