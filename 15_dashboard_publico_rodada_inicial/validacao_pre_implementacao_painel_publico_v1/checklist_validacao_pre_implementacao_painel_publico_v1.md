# Checklist de validação pré-implementação

## Status geral

- [x] Briefing de implementação consultado.
- [x] Arquitetura mínima consultada.
- [x] Nota metodológica pública consultada.
- [x] Decisão de uso público consultada.
- [x] Escopo documental preservado.
- [x] Nenhuma codificação realizada.

## Blocos obrigatórios

- [x] Abertura.
- [x] Universo analisado.
- [x] Principais sinais observados.
- [x] Operação.
- [x] Experiência.
- [x] Sinais condicionados.
- [x] Governança.
- [x] Limites.
- [x] Encaminhamento.

## Blindagens obrigatórias

- [x] Nota metodológica antes dos dados.
- [x] Sem rankings.
- [x] Sem atribuição pública de causa.
- [x] Sem exposição de perfis, links ou comentários identificáveis.
- [x] Território e dinâmica urbana fora dos eixos autônomos.
- [x] Linguagem sensível exige validação humana.

## Pendências bloqueantes

- [ ] Aprovar nota final.
- [ ] Aprovar recorte público.
- [ ] Aprovar sinais agregados.
- [ ] Aprovar campos de operação.
- [ ] Aprovar campos de experiência.
- [ ] Decidir uso ou não do Bloco 6.
- [ ] Aprovar forma de exibição de governança.
- [ ] Aprovar limites da leitura.
- [ ] Revisar linguagem pública final.
- [ ] Confirmar ausência de exposição identificável.
- [ ] Autorizar explicitamente codificação.

## Critério de aprovação

Enquanto qualquer pendência bloqueante estiver aberta, o status permanece `não_liberado_para_codar_ainda`.

## Registro operacional

- Dados alterados: 0.
- Matrizes existentes alteradas: 0.
- Painéis alterados: 0.
- Reprocessamento realizado: não.
