## Campos recomendados para operação

- fila;
- entrada;
- bar;
- banheiro;
- circulação;
- som;
- acesso;
- estrutura;
- organização.

## Critério de uso para operação

Os campos de operação devem representar funcionamento concreto da experiência. Cada campo só deve entrar na versão pública se estiver sustentado pelos dados disponíveis e puder ser apresentado em leitura agregada.

## Campos recomendados para experiência

- clima;
- vibe;
- encantamento;
- frustração;
- acolhimento;
- desejo de retorno;
- valor percebido;
- experiência geral.

## Critério de uso para experiência

Os campos de experiência devem representar percepção vivida do público. Cada campo só deve entrar na versão pública se estiver sustentado pelos dados disponíveis e puder ser apresentado como percepção pública agregada.

## Regras de blindagem

- não usar campo como ranking;
- não usar campo como prova de causa;
- não associar campo a responsável público;
- não expor perfis, links, autores ou comentários identificáveis;
- não criar eixo autônomo de Território e dinâmica urbana.

## Bloqueio

A seleção final dos campos ainda precisa de validação humana antes de qualquer codificação.
